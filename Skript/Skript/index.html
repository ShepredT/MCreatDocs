    如何制作脚本?

    只需要将插件放入服务端的plugins文件夹中重启服务器即可生成对应的配置文件与示例脚本文件.通常脚本文件是在scripts文件夹内的. 生成的示例文件带有 - 前缀，这些脚本并不会被加载运行.需要将前缀-删除后方可运行示例脚本.

    现在让我们制作第一个脚本

    先创建一个 file_name.sk格式的文件 file_name 是指文件名称. 这里演示取名为 hello.sk

    用文本编辑器打开,内容如下:

    # 当玩家每次加入游戏时,则发送一条信息 "Welcome to skript!"
on join:
    send: "Welcome to skript!"

    保存文件内容并将 hello.sk 放入 ..\plugins\Skript\scripts 文件目录下。

    你的第一个脚本已经完成!

    如何加载运行脚本？

    保存脚本的 sk 文件时，不会加载脚本。相反，它们在服务器启动时加载，或者在使用命令时加载：‎/sk reloa

    /sk reload hello # 重新加载名为 hello 的脚本
/sk reload subFolder/hello.sk # 重新加载子文件中名为 hello.sk 脚本
/sk reload subFolder/ # 重新加载子文件中的所有脚本
/sk reload scripts # 重新加载所有脚本

    现在可以尝试下,使用reload重新加载插件进行运行。
