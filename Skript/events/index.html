    时间事件

    Patterns: at %time% [in %worlds%]
    Since: 1.3.4

    在特定时间(游戏时间)调用

    at 18:00
at 7am in "world"

    On AoE Cloud Effect

    Patterns: [on] (area|AoE) [cloud] effect
    Since: 2.2-dev21

    Called when area effect cloud applies its potion effect. This happens every 5 ticks by default.

    on area cloud effect:

    动作事件

    Patterns: [on] [player] arm swing

    当玩家角色做出有关于手臂的相关动作时触发

    on arm swing:
    send "You swung your arm!"

    Patterns: [on] [player] armor change[d]

    On Block Damage

    Patterns: [on] block damag(ing|e)
    Since: 1.0

    当玩家破坏方块时调用。通常可以使用 leftclick 事件

    on block damaging:
    # 判定破坏方块的类型是否为 log (原木) - 当然可以是其他方块
    if block is log:
        send "你正在破坏原木方块"

    On Block Growth

    Patterns: [on] (plant|crop|block) grow[(th|ing)] [[of] %item types%]

    broadcast "plant=植物|crop=作物|block=方块|grow拥有两种模式 -> growth 或 gorwing"

    On Break/Mine

    Patterns: [on] [block] (break[ing]|min(e|ing)) [[of] %item types/block datas%]
    Since: 1.0 (break), unknown (mine), 2.6 (BlockData support)

    当方块被玩家摧毁时调用。如果使用 'Mine' , 只有破坏方块掉落事件才会调用。

    #---------------------------------------------#
# 第一种形式：未指定方块类型
# 特性一：当挖掘等级不符合则不调用，挖掘等级高于自身的方块时也不掉落
# 特性二：破坏完成之后调用。有一个先后顺序
on mine:
    broadcast "[on mine] 被调用"
#---------------------------------------------#
# 第二种形式：指定方块类型
on mine of any ore:
    broadcast "[on mine of any ore] 被调用"
#---------------------------------------------#
# 破坏 指定物品时才掉落 这里案例用的是 stone
on break of stone:
    broadcast "我破坏了 %item%"
#---------------------------------------------#
# 该定义了破坏箱子时,需要指定朝向
on break of chest[facing=north]:
    broadcast "箱子被破坏"
#---------------------------------------------#
# 该定义了破坏作物方块时，可指定作物的成熟度
on break of potatoes[age=7]:
    broadcast "作物被破坏"

    On Bucket Empty

    Patterns: [on] bucket empty[ing]
    [on] [player] empty[ing] [a] bucket
    Since: 1.0

    当清空水桶时调用. 可以使用 放置事件 来判定是水或者岩浆.

    on bucket empty:
    broadcast "&a案例：&7[&bOn Bucket Empty&7] &a成功调用！"

    On Bucket fill

    Patterns: [on] bucket fill[ing]
    [on] [player] fill[ing] [a] bucket
    Since: 1.0

    当玩家的桶中有液体时调用

    on player filling a bucket:
    broadcast "&a案例：&7[&bOn Bucket fill&7] &a成功调用！"

    On Burn

    Patterns: [on] [block] burn[ing] [[of] %item types/block datas%]
    Since: 1.0, 2.6 (BlockData support)

    当方块被烧毁时调用

    # 方块被点燃时并不会调用，而是方块被烧毁之后才调用！
# 注意顺序问题
on burn:
    broadcast "&a案例：&7[&bOn Burn&7] &a成功调用！"
# 可以指定多个物品类型 wood = 木头 ， fences = 栅栏 ， chests = 箱子
# 测试无效
on burn of wood, fences, or chests:
    broadcast "&a案例：&7[&bOn Burn&7] &a成功调用！"
on burn of oak_log[axis=y]:

    On Can Build Check

    Patterns: [on] [block] can build check
    Since: 1.0 (basic), 2.0 ([un]cancellable)

    当玩家持有方块或可放置物品时右键点击放置方块时调用. 可通过取消事件(cancel)来阻止放置方块,也可以取消.

    注意: 要放置方块的 data 是无法使用的.只有 ID 是有效可用的.

    on block can build check:
    cancel event

    On Chat

    Patterns: [on] chat
    Since: 1.4.1

    每当玩家发送消息时调用. 使用 聊天格式化 修改消息格式t, 使用 聊天对象 编辑收/发消息的聊天对象.

    on chat:
    if player has permission "owner":
        set chat format to "<red>[player]<light gray>: <light red>[message]"
    else if player has permission "admin":
        set chat format to "<light red>[player]<light gray>: <orange>[message]"
    else: #default message format
        set chat format to "<orange>[player]<light gray>: <white>[message]"

    On Command

    Patterns: [on] command [%text%]
    Since: 2.0

    当玩家输入命令时调用。

    on command:

    On Connect

    Patterns: [on] [player] connect[ing]
    Since: 2.0

    当玩家连接服务器时调用。

    与 on join 不同，该事件是指玩家在连接过程中时调用。

    on join 是玩家加入了服务器后才会被调用的。

    on connect:
    player doesn't have permission "VIP"
    number of players is greater than 15
    kick the player due to "The last 5 slots are reserved for VIP players."

    On Enderman/Sheep/Silverfish/Falling Block

    Patterns: [on] enderman place
    [on] enderman pickup
    [on] sheep eat[on] silverfish enter
    [on] silverfish exit
    [on] falling block land[ing]
    Since: unknown, 2.5.2 (falling block)

    当方块状态被改变时调用。

    末影人搬运方块或放置方块时、羊吃草地时、蠹虫钻入或钻出方块时、方块坠落落地形成方块状态时（沙子、沙砾）

    broadcast "A sheep stole some grass!"
on falling block land:
    if event-entity is a falling dirt:
        cancel event

    On Experience Spawn

    Patterns: [on] [e]xp[erience] [orb] spawn[on] spawn of [a[n]] [e]xp[erience] [orb]
    Since: 2.0

    当经验即将生成时调用.可以通过该事件取消经验生成. Please note that it's impossible to detect xp orbs spawned by plugins (including Skript) with Bukkit, thus make sure that you have no such plugins if you don't want any xp orbs to spawn. (Many plugins that only change the experience dropped by blocks or entities will be detected without problems though)

    on xp spawn:
    world is "minigame_world"
    cancel event

    On Hand Item Swap

    Patterns: [on] swap[ping of] [(hand|held)] item[s]
    Since: 2.3

    当玩家使用快捷键切换主副手槽内的物品时调用. 当然主副手的槽可以是空同样能调用. 该事件是在切换之前调用的, 这使你可以取消该事件的发生.

    on swap hand items:
    event-player's tool is a diamond sword
